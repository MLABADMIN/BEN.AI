# BEN.AI Phase 1 Launch QA Checklist

This checklist keeps the July 1st launch focused on the public BEN.AI guide experience, not the full future MLAB operating system.

## Brand and first impression

- BEN.AI introduces himself clearly.
- The visitor understands that BEN.AI belongs to MLAB / MyLifeAsBen.
- The experience uses the core language: The Adventure Starts Here, Built from real experiences, Designed for real life.
- The visual direction stays aligned with luxury black and gold.
- BEN.AI does not feel like a generic chatbot template.

## Core explanation coverage

BEN.AI should be able to explain:

- What MLAB is
- What BEN.AI is
- MLAB membership
- Travel services
- Relocation support
- Citizenship guidance
- Wealth Circles / Pardner Circles
- Community support concepts
- Business directory and opportunity sharing
- Life-admin support
- BEN.AI roadmap

## Audience readiness

BEN.AI should be useful for visitors, potential members, travel clients, relocation clients, investors, partners, affiliates and community members.

## Build and check process

Before launch, confirm:

- pnpm check passes.
- pnpm build passes.
- No obvious template branding remains in the public UI.
- README reflects BEN.AI / MLAB rather than the original chatbot template.
- Roadmap documentation is present.
- Prompt instructions align with the handover.

## Known follow-up items

- App metadata still needs final public-domain confirmation.
- Top header and template links should be rechecked visually.
- GitHub Actions failure logs should be reviewed directly if pnpm check still fails.
- BEN.AI portrait integration depends on the final approved image asset and layout direction.
- Homepage placement should keep MLAB as the main site and BEN.AI as the guide.
